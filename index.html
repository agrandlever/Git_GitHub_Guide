GitとGitHubの違い

Git

ファイルの変更履歴を記録するためのバージョン管理ツールです。

GitHub

Gitで管理したプロジェクトをオンラインで共有・共同作業できるサービスです。

Repository

プロジェクトのファイルと変更履歴をまとめて管理する場所です。

基本の流れ

  1. 1

    リポジトリを作る

    GitHubでプロジェクト用のリポジトリを作成します。

  2. 2

    ファイルを編集する

    HTML、CSS、JavaScriptなどのファイルを作成・編集します。

  3. 3

    コミットする

    変更内容にメッセージを付けて履歴として保存します。

  4. 4

    GitHubにプッシュする

    ローカルの変更をGitHub上のリポジトリに反映します。

よく使うコマンド

git status
git add .
git commit -m "Add first website"
git push origin main
